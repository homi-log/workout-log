# Google 시트 클라우드 동기화 설정

운동일지를 여러 기기에서 자동 동기화하려면, 본인 Google 계정의 스프레드시트를
저장소로 쓰도록 한 번만 설정하면 됩니다. (무료)

## 1. Google 시트 만들기

1. https://sheets.new 접속 → 빈 스프레드시트가 생성됩니다.
2. 이름을 알아보기 쉽게 지정 (예: `운동일지 데이터`).

## 2. Apps Script 붙여넣기

1. 상단 메뉴 **확장 프로그램 → Apps Script** 클릭.
2. 편집기에 있는 기본 코드(`function myFunction(){}`)를 전부 지웁니다.
3. 같은 폴더의 [`Code.gs`](Code.gs) 내용을 **전체 복사해서 붙여넣기**.
4. 💾 저장 (Ctrl+S).

## 3. 웹 앱으로 배포

1. 오른쪽 위 **배포(Deploy) → 새 배포(New deployment)**.
2. 톱니바퀴(유형 선택) → **웹 앱(Web app)** 선택.
3. 설정:
   - **설명**: 아무거나 (예: v1)
   - **실행 계정(Execute as)**: `나(Me)`
   - **액세스 권한(Who has access)**: **`모든 사용자(Anyone)`** ← 꼭 이걸로
4. **배포** 클릭.
5. 처음이면 **권한 승인** 창이 뜹니다:
   - 본인 Google 계정 선택 →
   - "이 앱은 Google에서 확인하지 않았습니다" 경고가 나오면
     **고급(Advanced) → (안전하지 않음) 이동** → **허용(Allow)**.
   - (내가 만든 스크립트이므로 안전합니다.)
6. 배포 완료 화면의 **웹 앱 URL**을 복사합니다.
   `https://script.google.com/macros/s/....../exec` 형태입니다.

## 4. 앱에 연결

1. 운동일지 앱(https://homi-log.github.io/workout-log/) 접속.
2. 오른쪽 위 **☁ 버튼** 클릭.
3. 복사한 URL을 붙여넣고 **연결 테스트** → "✓ 연결 성공"이 뜨는지 확인.
4. **저장 및 동기화** 클릭.
   - 이 기기에 데이터가 있으면 클라우드로 올라갑니다.
   - 이후 기록/수정은 자동으로 클라우드에 반영됩니다.
5. **다른 기기(PC 등)**에서도 같은 방법으로 ☁ → **같은 URL** 입력 → 저장.
   이제 모든 기기가 같은 데이터를 공유합니다.

## 동작 방식 / 참고

- 데이터 원본은 시트의 숨김 탭 `_db` A1 셀에 JSON으로 저장됩니다.
- `기록` 탭에 사람이 보기 좋은 표가 자동 생성됩니다(보기 전용 — 직접 편집해도 앱엔 반영 안 됨).
- 앱은 저장 시 **로컬(기기)에도 항상 캐시**하므로, 인터넷이 없어도 조회는 가능하고
  다시 연결되면 다음 저장 때 클라우드에 반영됩니다.
- 여러 기기에서 **동시에** 편집하면 나중에 저장한 쪽이 전체를 덮어씁니다
  (개인 용도에선 사실상 문제 없음). 한 기기에서 편집 후 다른 기기는
  ☁ → "클라우드에서 새로고침"으로 최신 상태를 받아오세요.

## 보안

- 웹 앱을 "모든 사용자"로 배포하지만, **URL을 아는 사람만** 접근할 수 있습니다.
- 이 URL은 앱 코드나 GitHub 저장소에 저장되지 않고 **각 기기의 브라우저에만** 저장되므로
  사실상 비공개입니다. (URL을 남에게 공유하지 마세요.)
- 더 강한 보안이 필요하면 요청 시 토큰 검증을 추가할 수 있습니다.

## 코드를 바꿨을 때 (재배포)

`Code.gs`를 수정했다면 **배포 → 배포 관리 → (연필) → 버전: 새 버전 → 배포**로
기존 배포를 업데이트하세요. URL은 그대로 유지됩니다.
